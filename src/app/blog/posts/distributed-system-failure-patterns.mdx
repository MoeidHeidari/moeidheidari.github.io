Distributed systems fail in predictable ways. Reliability improves when teams design for those failure modes from the beginning.

## Common patterns

- Network partitions
- Retry storms
- Thundering herds
- Cascading timeouts

## Mitigations

- Circuit breakers for unstable dependencies
- Exponential backoff with jitter
- Timeouts per dependency class
- Bulkheads to contain blast radius

## Testing approach

Run game days and controlled fault injection in pre-production. Measure recovery time and correctness under stress.

## Closing

Reliability is not a feature toggle. It is a continuous engineering practice.
